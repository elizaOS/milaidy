# Session 2026-03-02-002: Fix Desktop App API Connection + Electrobun Type Shim

## Summary
Fixed the Electrobun desktop app failing to connect to the backend API server. Root cause: WebSocket-based `executeJavascript()` silently drops messages when no client is connected (race condition on page load). Added three-layer injection using Electrobun's native APIs. Also rewrote the type shim to match the real Electrobun API docs with zero `any` types.

## Root Cause
`executeJavascript()` in `ipc-server.ts` sends eval messages over WebSocket, but on app startup the CEF webview hasn't connected yet â€” messages are silently dropped. The periodic retry only triggers on port *change*, so once set it never retries.

## Electrobun API Reference (from docs)
- Bun side: `electrobun/bun` â€” BrowserWindow, BrowserView, Utils, Tray, etc.
- Browser side: `electrobun/view` â€” Electroview class, defineRPC
- Native IPC: `BrowserView.defineRPC()` (bun) + `Electroview.defineRPC()` (browser)
- Preload IPC: `window.__electrobunSendToHost(message)`
- Globals in webview: `window.__electrobunWebviewId`, `window.__electrobunWindowId`
- Draggable regions: CSS classes `electrobun-webkit-app-region-drag` / `-no-drag`
