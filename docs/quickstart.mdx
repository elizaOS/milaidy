Use this quickstart for the shortest path to a working Milady setup.

## 1) Start Milady

```bash
milady
```

## 2) Complete onboarding

Typical prompts:

1. Agent name
2. Style/personality preset
3. Model provider + API key
4. Optional wallet setup

You can skip provider/wallet and configure them later.

## 3) Open dashboard

After startup, Milady prints local URLs. Open the dashboard URL in your browser.

## 4) Send first test message

Send a simple prompt such as:

```text
hello
```

Confirm you receive a model response.

## 5) Verify provider status

```bash
milady models
```

If provider is not configured correctly, fix credentials and retry.

## 6) Know the core commands

```bash
milady                    # interactive mode
milady start --headless   # service mode
milady dashboard          # open UI
milady doctor             # diagnostics
milady plugins list       # installed plugins
```

## 7) Validate local state paths

Milady stores state in:

- `~/.milady/milady.json`
- `~/.milady/workspace/`
- `~/.milady/logs/`
