This page is the API reference entrypoint. Endpoint details are split into domain pages under `/rest/*`.

## Base URL and transport

Default local API base:

```text
http://127.0.0.1:2138
```

Primary transports:

- REST (HTTP JSON)
- Streaming endpoints (SSE where applicable)

## Authentication model

When API auth is enabled, include bearer token:

```http
Authorization: Bearer <token>
```

Auth-related endpoints are documented at:

- `/rest/auth`
- `/rest/system`

## First-call integration workflow

1. Check service health/status
2. Confirm auth requirements
3. Send a minimal chat/conversation request
4. Inspect response/stream handling
5. Add domain-specific calls (config/plugins/connectors/etc.)

## API usage guidance

- Prefer explicit error handling for non-2xx responses.
- Treat streaming endpoints as long-lived connections.
- Avoid assumptions about optional fields; follow endpoint contracts.
- Keep tokens out of logs and screenshots.
